# ⛳ ゴルフスコア アプリ

ブラウザで動くゴルフスコア管理アプリです。  
URLをシェアするだけで仲間とリアルタイムにスコアを共有できます。

---

## 特徴

- **オフライン対応** — ゴルフ場の通信が悪くても入力継続できます
- **リアルタイム同期** — 同じURLを開いた全員のスコアが即座に反映されます
- **インストール不要** — ブラウザで開くだけで使えます（スマホ対応）
- **CSV出力** — ラウンド終了後にExcelで開けるCSVをダウンロードできます
- **コースライブラリ** — よく使うコースのPAR設定を保存・共有できます

---

## 画面構成

```
① メンバー  ② カウント  ③ スコア
```

管理者は `?admin` URLから管理者画面を利用できます。

---

## ① メンバー画面

### ラウンド共有
- アプリを開くと自動でセッションURLが生成されます
- **「日付」「ラベル」** を事前に設定しておくことができます
  - 日付：デフォルトは今日の日付。ラウンド予定日に変更可能
  - ラベル：ラウンド名を入力（例：春季コンペ 第2回）。最大30文字
  - 設定内容は同じURLを開いた全員にリアルタイムで同期されます
- **「URLをコピー」** でシェア用テキストをクリップボードに取得
  - ラベル・日付が設定済みの場合、URLの上にそれらが付加された形でコピーされます
  - LINEや メッセージにそのまま貼り付けるだけで、相手に内容が伝わります
  ```
  例）【春季コンペ 第2回】 2026-05-16
      https://our-golf-score-check.netlify.app/?session=xxxx
  ```
- 同じURLを開いた全員がリアルタイムでデータを共有
- **「新しいラウンド」** で新しいセッションを開始（ホール数・PARは18H/PAR4にリセット、メンバーもリセット。過去データは元のURLから参照可能）

### コース選択
- 検索欄にコース名を入力すると保存済みコースが絞り込まれます（部分一致）
- 検索欄が空の場合は保存済みコース一覧が表示されます

| ホール設定 | コース | ボタン | 動作 |
|-----------|--------|--------|------|
| 18ホール | 18Hコース | **適用** | 全18ホールのPARを適用 |
| 9ホール | 18Hコース | **OUT** / **IN** | 前半（1〜9番）または後半（10〜18番）のPARを9H設定で適用 |
| 9ホール | 9Hコース | **適用** | 9ホールのPARを適用 |

- ホール数・PARを設定後、**「現在の設定を保存」** にコース名を入力して保存できます
  - 保存したコースは仲間全員で共有されます
- コースを誤って登録した場合は **「削除依頼」** ボタンで管理者に削除を依頼できます（実際の削除は管理者が行います）

### ホール数
- **9ホール** または **18ホール** を選択
- ホール数を変更するとコース一覧のボタン表示も自動で切り替わります

### PAR設定
- ホールごとにPARを設定します（PAR 3〜5）
- 18ホールの場合はOUT（1〜9番）とIN（10〜18番）に分けて表示されます

### メンバー登録
- 名前を入力して **「追加」** ボタン（またはEnterキー）で登録
- 最大文字数：10文字、同名のメンバーは登録できません
- **「削除」** ボタンでメンバーとそのスコアを削除

### データ管理
| ボタン | 動作 |
|--------|------|
| スコアのみクリア | スコアだけ削除（メンバー・PAR設定は残る） |
| 全データをリセット | メンバー・スコア・設定をすべて削除 |

## ② カウント画面

### 基本操作
1. **メンバータブ** で入力するメンバーを選択
2. **ホールボタン**（丸い数字）で対象ホールを選択
3. **「＋」「−」ボタン** で打数を調整
4. **「決定 ›」ボタン** でスコアを確定し次のホールへ進む

### カウンターの仕様
- 初期表示はそのホールのPAR数
- **最小1打**（−ボタンを押し続けても1打より小さくはなりません）
- **「＋」「−」** を押した時点でスコアは即座に保存されます（「決定」は必須ではありません）
- **「決定 ›」** をタップすると、未入力の場合はPARとして記録し、次のホールへ移動します
- 入力済みのホールは丸ボタンに **緑の点** が表示されます
- 数字部分をタップすると直接入力モードになります（キーボードが表示されます、入力範囲：1〜20）

### スコア表示
| 表示 | 意味 |
|------|------|
| Hole-in-one | 1打でホールイン |
| Condor | −4（PAR5で1打など） |
| Albatross | −3 |
| Eagle | −2 |
| Birdie | −1 |
| Par | ±0 |
| Bogey | +1 |
| Double Bogey | +2 |
| （表示なし） | +3以上 |

### 接続状態
ヘッダーの **●** でオンライン状態を確認できます。
- 🟢 緑 — オンライン（リアルタイム同期中）
- 🔴 赤 — オフライン（入力データはローカルに保存、再接続時に自動同期）

## ③ スコア画面

- 縦列：メンバー、横行：ホール
- 各セルに打数と **PAR差（+/−）** を表示
- 18ホールの場合はOUT・IN小計も表示
- 最終行に合計スコアとPAR差を表示

### ラウンド確定
- **「🏁 ラウンドを確定する」** ボタンでラウンドの完了を宣言できます
- 確定後は確定日時が表示されます（スコアの修正は引き続き可能）
- 確定状態は管理者画面のラウンド一覧にも「🏁 確定済み」として反映されます
- メンバーが登録されていない場合はボタンは表示されません

### スコアの色
| 色 | 意味 |
|----|------|
| 紫 | Eagle以上 |
| 赤 | Birdie |
| 緑 | Par |
| 黄 | Bogey |
| オレンジ | Double Bogey |
| 濃い赤 | +3以上 |

### CSVダウンロード
**「CSVダウンロード」** ボタンでスコア表をCSV形式で保存できます。
- ファイル名：`golf-score-YYYYMMDD.csv`
- 日本語Excel対応（UTF-8 BOM付き）
- ホール別スコア・OUT/IN小計（18H）・合計を含む

## ⚙ 管理者画面

`?admin` をURLに付けて開きます（例：`https://example.com/?admin`）。  
開くとパスワード入力が求められます。認証はタブを閉じるまで有効です。  
**管理者URLは関係者以外にシェアしないでください。**

### 📋 ラウンド管理
- 全ラウンドの一覧と各ステータスを確認できます

| ステータス | 意味 |
|-----------|------|
| ⚪ 空 | メンバー未登録 |
| 🟡 準備中 | メンバー登録済み・スコア未入力 |
| 🟢 進行中 | スコア入力中 |
| ✅ 完了 | 全ホール入力済み |
| 🏁 確定済み | ユーザーがラウンド確定操作を実施済み |

- ラベルを入力してラウンドに名前を付けられます（例：「春季コンペ 第2回」）
- チェックボックスで複数選択して一括削除できます
- **「スコアを開く」** でそのラウンドのスコア画面を別タブで開けます

### 🏌️ コース管理
- 登録済みコースの一覧とホールごとのPAR詳細を確認できます
- ユーザーから **削除依頼** が届いたコースには「🗑 削除依頼あり」バッジが表示されます
  - **「削除」** ボタンでコースを完全削除
  - **「依頼却下」** ボタンで依頼フラグを解除（コースは残す）

### 📥 書き出し
- **全ラウンド一覧 CSV** — 全ラウンドのスコアを1ファイルに出力
- **個人スタッツ CSV** — メンバー別のラウンド数・平均スコア・ベストスコアを出力

### 📊 スタッツ
- メンバー横断でラウンド数・平均スコア・ベストスコア・PAR差を集計します
- 全ホール入力済みのラウンドのみ対象です

### 💾 DB使用量
- Firebase ストレージの推定使用量をグラフで確認できます
- 正確な値はFirebase Console でご確認ください

## オフライン時の動作

| 状況 | 動作 |
|------|------|
| 通信が途切れた | 入力はローカルに保存されるため継続可能 |
| 再接続した | 自動でFirebaseに同期される |
| ページを再読み込みした | ローカルのデータが表示される（Firebaseと同期後に最新データに更新） |

---

## 技術仕様

| 項目 | 内容 |
|------|------|
| フロントエンド | HTML / CSS / JavaScript（単一ファイル） |
| リアルタイムDB | Firebase Realtime Database |
| オフライン保存 | localStorage |
| ホスティング | Netlify |
| セッション管理 | URLパラメータ（`?session=UUID`） |
| コースデータ | Firebase共有DB（全ユーザーで共有） |

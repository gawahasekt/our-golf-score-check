Static Hosting Comparison 2026
Cloudflare Pages vs Netlify vs GitHub Pages
静的サイト・SPAホスティングの3大サービスを徹底比較。無料枠・パフォーマンス・セキュリティ・拡張性を網羅。
☁ Cloudflare Pages ◈ Netlify ◉ GitHub Pages
⚠️ 2025年9月〜 Netlify が新料金体系に移行（要注意）
2025年9月4日以降に作成された新規アカウントは「クレジット制」に変更。無料枠は 300クレジット/月 で、本番デプロイ1回に15クレジットを消費するため 実質20回/月でビルド上限 に達する。頻繁に更新するアプリでは想定外にクレジットが枯渇する。
💡 この比較について
2026年5月時点の各サービス公式ドキュメントをもとに作成。料金・仕様は変更される場合があります。

# Netlify 新クレジット制の計算例

無料枠: 300 クレジット / 月

消費レート:
  本番デプロイ 1回     → 15 クレジット
  帯域幅 1GB           → 20 クレジット
  Functions 1GB時間    → 10 クレジット

# デプロイのみで試算
  300 ÷ 15 = 月 20回 デプロイでビルド上限到達
  帯域 5GB 消費した場合: 残り 300 - 100 = 200c → 13回分のみ

🏆 結論・こんな用途に向いている
🥇
Cloudflare Pages
総合最強・推奨
無制限帯域幅・世界最高水準のCDN・DDoS防御・Workers によるサーバーサイド処理と、すべての面でバランスが取れている。Netlifyのクレジット問題を根本解決できる移行先として最適。設定ファイルの構文もNetlifyとほぼ同じため移行コストも低い。
  • コスト永続ゼロで帯域を気にせず使いたい
  • セキュリティヘッダー・DDoS防御を無料で使いたい
  • 将来的にサーバーサイド処理（API・認証等）を追加したい
  • Netlifyから移行したい（設定ファイルをそのまま使える）
  • 日本向けサービスで低レイテンシを重視したい
SPAアプリ 認証付きサービス 本番運用 Netlify移行先
🥈
Netlify
DX最優秀・有料なら快適
開発体験・ドキュメント・エコシステムの充実度は3サービス中トップ。ただし新規アカウントの無料枠がクレジット制になり、頻繁にデプロイするプロジェクトでは月20回で上限到達という厳しい制約がある。有料プラン（$20/月〜）に移行するか、Cloudflareへ乗り換えを検討したい。
  • Deploy Previews で PR ごとの確認を徹底したい
  • 日本語情報・コミュニティを活用したい
  • Pro プランを支払っても最高の開発体験が欲しい
チーム開発 フルスタックアプリ 有料プラン利用
🥉
GitHub Pages
シンプル・ドキュメント特化
完全無料・設定最小でとにかく公開したいケースに最適。ただし、セキュリティヘッダー設定不可・サーバーサイド処理なし・プレビューデプロイなしという制約は大きく、本格的なWebアプリには向かない。OSSのドキュメントサイト・ポートフォリオのような静的コンテンツが最も適している。
  • GitHub にソースコードがあり、すぐ公開したい
  • 拡張性より「とにかく無料・簡単」を優先したい
  • セキュリティヘッダー等の制約を許容できる
